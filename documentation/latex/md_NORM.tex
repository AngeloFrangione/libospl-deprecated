This norm aims to make a code that is coherent and readable to other contributors. ~\newline
 You should follow these rules as much as you can. ~\newline
 Not following formatting rules will likely make your pull request rejected. ~\newline
 Doing forbidden things will make us reject your pull request. ~\newline
 If you have any question or remark about this norm, open an issue. We will read it carefully.\hypertarget{md_NORM_autotoc_md13}{}\doxysection{Part 1\+: naming}\label{md_NORM_autotoc_md13}
Naming variables, functions names and other stuff like file names must follow these guidelines\+:


\begin{DoxyItemize}
\item Names must be clear and in english and as explicit as possible (except for counters)
\item Counters with explicit names are appreciated (l for line counter, c for column counter, ...)
\item Only lower case ascii characters allowed
\item Words must be separated with underscore \textquotesingle{}\+\_\+\textquotesingle{} (ex\+: {\ttfamily my\+\_\+var})
\item Some data structures must be prefixed like this \+:
\end{DoxyItemize}

The code itself should respect a few things\+:


\begin{DoxyItemize}
\item You must indent the code with . \+\_\+\+\_\+\+\_\+tabulations\+\_\+\+\_\+\+\_\+ not {\itshape spaces}.
\item Try to avoid making functions over 30-\/35 lines. Try to {\itshape split} the code {\bfseries{as most as you can}}. Avoid making functions that can be done \+\_\+\+\_\+\+\_\+reÃ©usable\+\_\+\+\_\+\+\_\+.
\item Try to avoid writing lines that are larger than 100 columns. Do not forget that tabulations are 4 columns large.
\item Try to get the smallest form of your conditions. Smaller conditions are often easier to understand and optimized.
\item Only one instruction per line.
\item An empty line is an {\itshape empty} line. No spaces, no tabulations. Empty.
\item Declaring variable on the top of the function is often clearer but not mandatory.
\item Try to space out your code sensibly. Avoid using more than one empty line.
\item The end of a line should be empty. {\itshape Empty} like empty lines.
\item You must put the curly braces at the next line\+:
\end{DoxyItemize}

\hypertarget{md_NORM_autotoc_md16}{}\doxysection{Part 4\+: files.\+c}\label{md_NORM_autotoc_md16}

\begin{DoxyItemize}
\item File names must be explicit about what they contain. ~\newline
 If it contains only a struct definition, it should have the struct name as its own name. If it conatins only one function, it should have the function name as its own name.
\item A file should never contain functions that are not related one to each other.
\item Use static keyword when a function is reachable only in the file where it is defined.
\item Please write the short licence at the top of the file
\end{DoxyItemize}\hypertarget{md_NORM_autotoc_md17}{}\doxysection{Part 5\+: header.\+h}\label{md_NORM_autotoc_md17}

\begin{DoxyItemize}
\item Protect it against double inclusion. If the file is named name.\+h you should {\ttfamily \#define N\+A\+M\+E\+\_\+H}
\item These definitions must be in header files
\begin{DoxyItemize}
\item Function prototypes
\item Structure, union and enum
\item Typedefs
\end{DoxyItemize}
\item Defines are in uppercase (ex\+: {\ttfamily \#define R\+E\+D\+\_\+\+C\+O\+L\+OR \textbackslash{}033\mbox{[}0;31m})
\item Please write the short licence at the top of the file
